# 강아지 쓰다듬기 반응 이벤트 설계

작성일: 2026-05-24

## 목적

첫 강아지 프로토타입의 목표는 강아지가 플레이어 입력에만 움직이는
오브젝트가 아니라 살아있는 동반자처럼 느껴지는지 검증하는 것이다.

초기 검증 대상은 전체 강아지 상태 시스템이 아니라 쓰다듬기 상호작용이다.
첫날의 낯선 장소에서 조심스러워하던 강아지가 쓰다듬기를 통해 안정되는
순간을 작게 만든다.

에셋은 아직 준비되지 않았으므로, 현재 구현은 placeholder 표현을 사용해도
된다. 대신 나중에 실제 강아지 스프라이트, 얼굴 프레임, 애니메이션,
파티클 에셋을 붙일 때 게임 로직을 바꾸지 않아도 되는 구조를 목표로 한다.

## 배경

`docs/game/` 정본 문서는 이 게임이 강아지를 효율적으로 관리하는 게임이
아니라, 낯선 별나라에서 강아지와 함께 적응해가는 따뜻한 생활 게임이라고
정의한다.

이번 설계는 다음 정본 방향을 따른다.

- 강아지가 살아있는 존재처럼 느껴져야 한다.
- 밥 주기, 공놀이, 쓰다듬기는 서로 다른 감정으로 느껴져야 한다.
- 쓰다듬기는 강아지의 안정감을 회복하는 정서적 핵심 상호작용이다.
- 별나라의 신비함은 강아지와 보내는 하루를 특별하게 만드는 배경이어야 한다.

외부 자료로는 Exa 검색을 통해 초반 게임 설계 방향을 확인했다. Unity Learn의
프로토타입 가이드는 첫 프로토타입을 가장 기본 상호작용과 그 의존성으로
좁히라고 설명한다. GameDeveloper의 simulation-driven playable 글은 낮은
충실도라도 연결된 플레이어블 경험을 먼저 검증하라고 설명한다. MDA
프레임워크는 플레이어가 느낄 감정에서 출발해 동작과 규칙으로 내려가는
설계 관점을 제공한다.

## 결정 사항

- 첫 검증 목표는 "강아지가 살아있다는 느낌"이다.
- 첫 상호작용은 쓰다듬기다.
- 쓰다듬기 전 강아지는 낯선 곳이라 조심스러운 감정에서 시작한다.
- 쓰다듬기 중에는 표정이 편안하게 변하고 별나라식 별빛 이펙트가 발생한다.
- 이펙트의 역할은 세계관 피드백이다. 별나라가 강아지의 안정감에 반응하는
  느낌을 준다.
- 상태 머신을 먼저 크게 만들지 않는다.
- 게임 로직은 실제 에셋 이름이나 애니메이션 이름을 직접 알지 않는다.
- 게임 로직은 반응 이벤트 계약을 만들고, 표현 레이어가 placeholder 또는
  실제 에셋으로 변환한다.

## 하지 않을 것

- 전체 강아지 생활 AI를 만들지 않는다.
- 복잡한 기분, 욕구, 질병, 방치 페널티 시스템을 만들지 않는다.
- 최종 스프라이트, 애니메이션, 파티클 에셋을 전제로 설계하지 않는다.
- 강아지 상태 enum을 큰 중심 구조로 먼저 고정하지 않는다.
- 밥 주기와 공놀이까지 한 번에 설계하지 않는다.
- 플레이어에게 `calmness` 같은 내부 수치를 직접 보여주지 않는다.

## 아키텍처

구조는 네 역할로 나눈다.

```text
입력/상호작용 감지
-> 강아지 반응 결정
-> 표현 요청 생성
-> placeholder 또는 실제 에셋으로 표시
```

### `PetInteraction`

플레이어가 쓰다듬기 입력을 했는지 판단한다.

이 단위는 강아지의 감정, 애니메이션, 이펙트, 에셋 이름을 모른다. 입력이나
상호작용 범위가 유효하면 쓰다듬기 시작 또는 완료 이벤트를 만든다.

### `DogReactionSystem`

쓰다듬기 이벤트를 받아 어떤 감정 반응이 필요한지 결정한다.

예를 들어 조심스러운 표정에서 편안한 표정으로 바뀌는 요청, 별빛이 피어나는
이펙트 요청, 안정감 증가를 포함한 반응 요청을 만든다.

이 단위는 실제 에셋 이름을 모른다.

### `DogMood`

최소 감정 데이터만 가진다.

첫 버전에서는 `calmness` 정도만 둔다. 이 값은 큰 육성 시스템이 아니라
쓰다듬기 반응의 강도와 후속 idle 표현을 조절하는 작은 입력이다.

### `DogPresenter`

반응 요청을 화면 표현으로 바꾼다.

현재는 도형, 색 변화, 단순 얼굴 placeholder, 간단한 별빛 파티클로 표현한다.
나중에는 같은 반응 요청을 실제 강아지 얼굴 프레임, 애니메이션,
스프라이트 이펙트, 파티클 에셋에 매핑한다.

## 반응 이벤트 계약

반응 이벤트는 "무엇을 보여줘야 하는가"를 말한다. "어떤 최종 에셋을
재생할 것인가"는 말하지 않는다.

예상 형태는 다음과 같다.

```ts
{
  expression: "cautious_to_relaxed",
  effect: "starlight_bloom",
  intensity: "soft",
  moodDelta: { calmness: 1 }
}
```

각 필드의 의미는 다음과 같다.

- `expression`: 강아지 표정 또는 자세가 어떤 감정 방향으로 바뀌어야 하는지
  표현한다.
- `effect`: 별나라식 시각 반응을 표현한다.
- `intensity`: placeholder와 실제 에셋 모두에서 반응 강도를 조절하는 힌트다.
- `moodDelta`: 반응 이후 내부 감정값에 적용할 작은 변화다.

## 쓰다듬기 반응 흐름

첫 프로토타입의 흐름은 다음과 같다.

```text
1. 강아지는 조심스러운 기본 반응으로 시작한다.
2. 플레이어가 쓰다듬기 입력을 한다.
3. PetInteraction이 쓰다듬기 시작 이벤트를 만든다.
4. DogReactionSystem이 안정 반응 요청을 만든다.
5. DogPresenter가 표정 변화와 별빛 이펙트를 보여준다.
6. DogMood의 calmness가 조금 오른다.
7. 반응이 끝나면 강아지는 더 편안한 idle 표현으로 돌아간다.
```

표현 예시는 다음과 같다.

```text
쓰다듬기 전: 조심스러운 표정, 작고 약한 별빛
쓰다듬는 중: 편안해지는 표정, 털 주변에 별빛이 피어남
쓰다듬은 후: 더 부드러운 idle 표현, 안정된 분위기
```

## 에셋 연동 전략

에셋이 없을 때도 반응 계약은 그대로 사용한다.

```text
expression: 색, 눈 모양, 간단한 라벨 변화
effect: 작은 별빛 파티클 placeholder
calmness: 후속 표정과 이펙트 강도에만 영향
```

에셋이 생긴 뒤에는 `DogPresenter`에서만 매핑을 바꾼다.

```text
expression -> 강아지 얼굴 프레임 또는 애니메이션
effect -> 별빛 파티클 또는 스프라이트 이펙트
intensity -> 애니메이션 속도, 파티클 수, 밝기, 지속 시간
```

이 설계의 핵심 규칙은 다음과 같다.

```text
DogReactionSystem은 실제 에셋 이름을 모른다.
DogPresenter만 placeholder와 실제 에셋 매핑을 안다.
```

## 에러 처리와 fallback

첫 버전에서는 에셋 없음이 정상 상태다. 리소스 누락은 실패가 아니라
placeholder 표현으로 처리한다.

표현 요청 처리 순서는 다음과 같다.

```text
1. 실제 에셋 매핑이 있으면 실제 에셋을 사용한다.
2. 실제 에셋 매핑이 없으면 placeholder 표현을 사용한다.
3. 알 수 없는 expression은 기본 편안한 표정 placeholder로 처리한다.
4. 알 수 없는 effect는 no-op 또는 기본 별빛 placeholder로 처리한다.
```

실패로 봐야 하는 경우는 다음과 같다.

- 쓰다듬기 입력이 반응 시스템까지 전달되지 않는다.
- `moodDelta`가 적용되지 않는다.
- `DogPresenter`가 반응 요청을 받아도 아무 시각 변화가 없다.
- 알 수 없는 표현 요청이 런타임 오류를 발생시킨다.

## 테스트와 검증

자동 테스트는 렌더링 픽셀보다 계약 테스트를 우선한다.

- 쓰다듬기 시작 이벤트를 주면 `DogReactionSystem`이
  `cautious_to_relaxed` 반응 요청을 만든다.
- 쓰다듬기 반응이 적용되면 `calmness`가 증가한다.
- 알 수 없는 `expression` 또는 `effect` 요청도 `DogPresenter`가 오류 없이
  fallback한다.
- `DogReactionSystem` 테스트는 Phaser 렌더링이나 실제 에셋에 의존하지 않는다.

수동 확인은 다음 한 가지를 본다.

```text
쓰다듬기 입력 시 표정 placeholder가 바뀌고 별빛 placeholder가 발생하는가?
```

## 향후 확장

이 구조가 안정되면 같은 반응 이벤트 계약을 밥 주기와 공놀이에도 확장할 수
있다.

- 밥 주기: 조심스러움에서 안심으로 바뀌는 냄새 맡기와 먹기 반응
- 공놀이: 망설임에서 활기로 바뀌는 뛰기와 돌아오기 반응
- 하루 종료: 안정된 강아지가 가까운 위치에서 잠드는 반응

확장은 새 시스템을 크게 추가하기보다, 기존 반응 계약에 필요한 표현과 감정
변화를 작게 추가하는 방식으로 진행한다.
